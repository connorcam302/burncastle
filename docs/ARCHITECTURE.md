# Architecture

Burncastle is a SvelteKit app backed by Convex. The frontend renders the site, calls Convex queries for live data, and fires Convex mutations for auction actions.

## Stack

- SvelteKit 2
- Svelte 5 runes
- Vite 7
- Tailwind CSS 4
- Convex
- `convex-svelte`
- `@lucide/svelte` for icons
- `dayjs` is installed but not currently used in the scanned files

## App Structure

```text
src/routes
  +layout.svelte              Root shell, logo, navbar, Convex setup
  +page.svelte                Homepage tiles

  icons/*
  ui/*

  schema.ts
  auctions.ts
  bids.ts
  matches.ts
  players.ts
```

## Runtime Flow

`src/routes/+layout.svelte` imports `PUBLIC_CONVEX_URL` and calls:

```ts
setupConvex(PUBLIC_CONVEX_URL);
```

That makes Convex available to components using `useQuery` and `useConvexClient`.

Most data-heavy pages query Convex directly from Svelte components:

- `api.auctions.getAll`
- `api.auctions.get`
- `api.matches.getAll`
- `api.players.getAll`
- `api.bids.getOnePlayer`

Mutations are called with `client.mutation(...)` from the auction detail page.
The matches page also calls `api.matches.save` to create and update match records, teams, participants, player card stats, and timeline events.

## Styling

Global styling is in `src/app.css`.

Important theme conventions:

- Font: Maven Pro from Google Fonts.
- Burncastle accent: `--color-burnley: #5e003f`.
- Highlight: `--color-highlight: #ffd700`.
- Page background: `static/background.jpg`.
- Logo: `static/logo.png`.

The current visual language is dark, football-card inspired, uppercase, and gradient-heavy.

## Static Assets

Player portraits are expected at:

```text
static/players/{nameId}.png
```

Examples:

- `static/players/connor.png`
- `static/players/potto.png`
- `static/players/stevo.png`

Components build image URLs like:

```svelte
<img src={`/players/${player.nameId}.png`} />
```

Adding a player requires both a Convex `players` row and a matching image file.

## SSR Notes

`src/routes/auctions/[matchId]/+page.ts` sets:

```ts
export const ssr = false;
```

This is currently needed because the page reads `localStorage` and is highly client-side. Other routes may SSR by default.

## Generated Convex Files

Files under `src/convex/_generated` are generated by Convex. Do not hand-edit them. Regenerate through the Convex CLI when schema or function APIs change.
